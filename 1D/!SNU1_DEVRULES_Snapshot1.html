1D Mode development rules

Go back to the 1D home page

1D mode has a set of rules for development. These are in place to keep the simple style of the pages. The rules are pretty general.

Allowed HTML tags

The allowed HTML tags in 1D mode are:

< p > < / p > < P > < / P >

< i > < / i > < I > < / I >

< b > < / b > < B > < / B >

< details > < / details > < DETAILS > < / DETAILS >

< h1 > < / h1 > < H1 > < / H1 >

< h2 > < / h2 > < H2 > < / H2 >

< h3 > < / h3 > < H3 > < / H3 >

< h4 > < / h4 > < H4 > < / H4 >

< h5 > < / h5 > < H5 > < / H5 >

< h6 > < / h6 > < H6 > < / H6 >

< hr > < HR >

< html > < / html > < HTML > < / HTML >

< head > < / head > < HEAD > < / HEAD >

< a > < / a > < A > < / A >

< u > < / u > < U > < / U >

< s > < / s > < S > < / S >

< summary > < / summary > < SUMMARY > < / SUMMARY >

< body > < / body > < BODY > < / BODY >

< title > < / title > < TITLE > < / TITLE >

< bgcolor > < BGCOLOR >

< ! - - comment - - ! >

No other tags are allowed here. This is to keep the page as simple as possible, and also to let it run in any web browser. If you want to use different tags, try 2D or 3D mode

Allowed Media types

Even though this mode is text only, you can still reference some files. Just remember that images, videos, or audio files are not allowed

The allowed media types in 1D mode are:

.htm

.html

.txt

.exe

.iso

.rpm

.dmg

.img

.zip

.doc

.rtf

.bat

.bin

.odt

.tar

.7z

.tmp

No other file types can be references here. This mode is designed to be run on any browser, all the way down to Windows 3.1/System 7 (MacOS)/LINUX (unknown)/UNIX 7/etc

Again, if you want to use different media types, try 2D or 3D mode

Allowed colors

1D mode has very partial color support, with only 16 supported colors.

Here are the allowed colors:

Black (RGB: 0,0,0 HEX: 000000)

Dark red (RGB: 128,0,0 HEX: Unknown

Red (RGB: 255,0,0 HEX: FF0000)

Pink (RGB: 255,0,255 HEX: FFC0CB)

Teal (RGB: 0,128,128 HEX: Unknown

Green (RGB: 0,128,0 HEX: Unknown

Bright green (RGB: 0,255,0 HEX: Unknown

Turquoise (RGB: 0,255,255 HEX: 40E0D0

Dark blue (RGB: 0,0,128 HEX: Unknown

Violet (RGB 128,0,128 HEX: Unknown

Blue (RGB: 0,0,255 HEX: 0000FFF)

Gray 25% (RGB: 192,192,0 HEX: Unknown

Gray 50% (RGB: 128,128,128 HEX: Unknown

Dark yellow (RGB: 128,128,0 HEX: Unknown

Yellow (RGB: 255,255,0 HEX: Unknown

White (RGB: 255,255,255 HEX: Unknown

Also make sure what you make also looks good in 1 bit Monochrome, or Grayscale/Greyscale as it might be viewed that way

Again, if you want to use even more colors, try 2D or 3D mode

Page size

Since this page is meant to be viewed on slower browsers, the page size has to be a lot smaller

Make sure your page is within 2048 lines of code, no more than 512 characters per line, and has less than 65536 characters total

If needed, make a second page as a part 2, and link to it

If you want to make your page bigger than 65.536 Kilobytes in size, go to 2D or 3D mode

Allowed characters (English mode)

This mode is designed to be highly compatible with old and new browsers. For compatibility with early 1990s web browsers, the character set of this mode is limited to ASCII

The allowed characters are:

Numbers

0123456789

Symbols

@!$%#^&?+=

Punctuation

`~(){}[]-_|\/<>.,;:"'

Letters

aAbBcCdDeEfFgGhHiIJjKklLmMnNoOpPqQrRsStTuUvVwWxXyYzZ

If you want to use ANSI, UTF-1, UTF-8, UTF-16 and other encodings, try 2D or 3D mode

Allowed characters (other)

Unknown

Those are the rules for development in 1D mode
